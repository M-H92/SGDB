Plus lourd que la moyenne

Voici un petit challenge pour vous : pour la base de données fat_cat_clinic, écrire une query qui permettrait de récupérer les chats plus lourd que la moyenne.
À priori, il serait intéressant d'utiliser la table weighting qui contient les pesées des différents chats.
Instinctivement, vous pourriez certainement être tenté d'écrire quelque chose de ce genre : 

SELECT * FROM weighting
WHERE weight > AVG(weight)

Et vous auriez une très bonne intuition.

Néanmoins, rappelez-vous des contraintes des fonctions d'agrégation comme AVG. Elles ne peuvent pas être utilisées dans une clasue WHERE
Dans ce genre de situation, une SUBQUERY peut être utilisée.

Rappel : les fonctions d'agrégations seront utilisées avec des clauses HAVING plutot que WHERE

Subquery

Une subquery (en français, une requête imbriquée) est, comme son nom l'indique, une requête imbriquée à l'intérieur d'une autre requête.
Ceci permet de palier au problème d'utiliser des résultats de requêtes dans d'autres requêtes et peut, dans certains cas, être utilisé comme substitut aux jointures.
On pourra donc résoudre notre problème ci-dessus avec le code suivant : 

SELECT * FROM weighting
WHERE weight > (SELECT AVG(weight) FROM weighting);

Si l'on décortique cette requête, on se rend compte que les deux composants, séparés, sont assez simple.
D'un côté, la requête interne sélectionne uniquement la moyenne des pesées.
De l'autre côté, la requête externe sélectionne les pesées pour lesquelles les valeurs sont supérieures au résultat de la requête interne : la moyenne des pesées
